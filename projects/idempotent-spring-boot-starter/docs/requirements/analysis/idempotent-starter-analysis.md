# 幂等 Starter 需求分析

## 1. 背景

目标不是交付一个已经完成的组件，而是提供一个符合真实工程结构、可以逐步亲手完成的学习项目。

## 2. 范围

- 提供纯 Java 核心模块、Spring Boot Starter 和示例应用。
- 提供公共契约、自动配置、练习说明和可启用的契约测试。
- 第一阶段仅支持单进程内存存储。
- 核心仓储并发控制、执行状态流转和 SpEL 解析必须由学习者实现。

## 3. 需求澄清 Gate

状态：`NOT_REQUIRED`

原因：用户已明确技术语言、项目位置、交付边界和参与方式；第一阶段不涉及跨服务、数据库或外部接口。

## 4. 领域上下文

长期领域文档和 ADR：`NONE`。

## 5. 验收与排期

### 5.1 验收

- 在 JDK 21 下执行 `mvn test` 能完成编译和骨架测试。
- 示例项目能启动并加载 Starter 自动配置。
- 三处核心练习有明确 TODO、说明和默认禁用的契约测试。
- 未实现 Redis、结果复用、续租等第二阶段能力。

### 5.4 排期预估

> **唯一排期**维护在 Spec 文档中：`docs/design/spec/idempotent-starter-spec.md` 第5节
>
> 当前预估总计 **2.5d**（含 20% 缓冲），以 Spec 文档为准。
