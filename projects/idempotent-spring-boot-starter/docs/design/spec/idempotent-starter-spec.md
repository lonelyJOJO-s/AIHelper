# 1 需求分析

> 完整需求分析：`docs/requirements/analysis/idempotent-starter-analysis.md`
> 问题清单：`docs/requirements/analysis/idempotent-starter-issues.md`

## 1.1 背景

创建一个工程结构真实但保留核心练习的基础组件项目，让学习者亲手掌握幂等状态、并发竞争和 Spring 集成。

## 1.2 目标与非目标

**目标（Goals）**：
- 提供可编译、可启动、可逐步完成的多模块项目骨架。
- 通过契约测试引导学习者实现内存仓储、核心执行器和 SpEL Key 解析。

**非目标（Non-Goals）**：
- 第一阶段不接入 Redis、数据库或消息队列。
- 不实现结果缓存、自动续租、分布式锁和生产级监控。
- Codex 不代写三个核心练习的实现。

## 1.3 需求要点

- 项目位于 AIHelper 的 `projects/idempotent-spring-boot-starter/`。
- 公共 API 和 Spring Boot 接线完整，核心算法明确留白。
- 默认构建保持绿色，学习者逐个启用练习测试。

# 2 功能规格

## 2.1 功能点与验收标准

> 逐条列出功能点，给出验收标准（可量化、可自动验证），便于快速评审和验收。

| 序号 | 功能点 | 验收标准 | 问题/风险 |
| --- | --- | --- | --- |
| 1 | 多模块骨架 | JDK 21 下 `mvn test` 成功 | Maven/JDK 环境差异 |
| 2 | 核心公共契约 | 业务方可通过单一执行入口提交幂等命令 | 核心逻辑故意未实现 |
| 3 | Spring Boot Starter | 引入依赖后自动注册仓储、执行器、解析器和切面 | 自动配置兼容性 |
| 4 | 学习练习 | 三个核心练习均有 TODO、说明和独立契约测试 | 用户需要逐项启用测试 |
| 5 | 示例应用 | 应用可启动，示例方法带幂等注解 | 未完成练习前调用会失败 |

## 2.2 技术约束

> 约束是控制 AI 实现质量的核心手段。写清"必须用什么"和"不能用什么"，不写"怎么实现"。
